# ADR-004: Decoupling AI from Core Commerce

## Context
Nova Sphere integrates advanced LLM orchestration, vector search, and AI-driven personalization. These services depend heavily on external providers (OpenAI, Anthropic) and complex internal embedding infrastructure. If the AI layer experiences an outage, latency spike, or budget exhaustion, it must not impact the core ability of a customer to checkout.

## Decision
We mandate strict architectural decoupling between the AI subsystem and Core Commerce.
1. **Foreign Key Isolation**: AI entities (e.g., `ShoppingMemory`, `PersonalizationProfile`) may store `userId` or `productId`, but they must not enforce strict database-level referential integrity (foreign keys) that would cascade locks onto Commerce tables.
2. **Graceful Degradation**: The UI and API must explicitly catch AI timeouts and fallback to standard SQL-based search or generic recommendations.
3. **Budget Isolation**: AI usage is strictly gated by the `AIBudget` table, which sits entirely outside the order flow.

## Rationale
- Core Commerce (adding to cart, checkout, payment processing) is the lifeblood of the platform. Its availability must be > 99.99%.
- AI services inherently suffer from higher latency and external dependencies, making 99.99% availability impossible.
- Decoupling ensures that an OpenAI outage means "suboptimal product discovery," not "zero revenue."

## Consequences
- Product engineers must always implement a "dumb" fallback for every "smart" feature.
- AI memory cleanups or vector re-indexing can happen asynchronously without slowing down transactional databases.
