# ADR-004: Why AI Gateway

## Context
The platform heavily leverages Large Language Models (LLMs) for search, support, Q&A, and commerce orchestration. Relying on direct API calls to OpenAI or Anthropic throughout the codebase creates vendor lock-in, unmanaged costs, and severe security risks regarding prompt injection and PII leakage.

## Decision
We will implement a centralized **AI Gateway** as the single point of entry for all LLM traffic.

## Rationale
- **Provider Abstraction**: Allows seamless fallback routing. If OpenAI goes down or increases pricing, the gateway can seamlessly route traffic to Anthropic or local models.
- **Cost Controls**: Centralized enforcement of `AIBudget`. We can hard-stop requests if a tenant exceeds their monetary limit.
- **Security & Safety**: Centralized PII scrubbing and prompt injection detection before any data leaves our network.
- **Observability**: Every request generates a `PromptLog` for precise auditing, latency tracking, and ROI calculation.

## Consequences
- The AI Gateway becomes a critical path component; if it fails, all AI features degrade.
- Requires rigorous performance tuning to ensure it does not add unacceptable latency to the LLM round-trip.
