# ADR-003: Why Signals Engine

## Context
Nova Sphere requires profound user intelligence to power the discovery engine, dynamic UI personalization, and automated AI interventions. Traditional analytics tools (like Google Analytics) provide retrospective dashboards but fail to feed real-time intelligence back into the application logic.

## Decision
We will build a proprietary **Signals Engine** that captures, processes, and persists micro-interactions (hovers, dwell time, search queries, cart abandonments) directly into our backend.

## Rationale
- **Closed-Loop AI**: Telemetry is fed instantly into the `PersonalizationProfile` and `ShoppingMemory`, allowing the application to adapt the UI in real-time (e.g., displaying a discount if the user dwells on a checkout button).
- **Data Ownership**: We retain total control over the behavioral data, allowing us to train bespoke ranking models without third-party vendor constraints.
- **Contextual Search**: Signals inform the vector search engine to boost products based on immediate behavioral context, not just historical purchases.

## Consequences
- Capturing high-volume telemetry requires batching (e.g., saving to Redis first, then flushing to PostgreSQL `SignalsLedger`).
- We are responsible for adhering strictly to GDPR/CCPA data retention and anonymization rules.
