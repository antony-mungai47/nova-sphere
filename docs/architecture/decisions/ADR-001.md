# ADR-001: Why PostgreSQL

## Context
Nova Sphere requires a primary persistence store to replace the prototype in-memory `Map<string, any>` structures. The system handles highly transactional e-commerce workflows (orders, payments, inventory, auctions), complex relational data (catalog, vendor hierarchy), and unstructured data (AI prompts, telemetry).

## Decision
We have selected **PostgreSQL** (specifically managed via Neon) as the primary system of record.

## Rationale
- **ACID Compliance**: E-commerce ledgers and auction bids strictly require ACID guarantees to prevent race conditions and over-selling.
- **JSONB Support**: Allows us to store unstructured AI responses, metadata, and telemetry alongside strongly typed relational data without needing a separate NoSQL database.
- **Ecosystem**: Outstanding support via Prisma ORM.
- **Branching**: Neon's branching allows for isolated ephemeral databases per PR, completely satisfying our CI/CD integration testing requirements without database mocking.

## Consequences
- Requires strict schema management and migrations.
- We must enforce the Transactional Outbox pattern to safely integrate PostgreSQL with our asynchronous message buses.
