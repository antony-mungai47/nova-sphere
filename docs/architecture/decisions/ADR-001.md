# ADR-001: State Machine over DAG for Workflow Automation

## Context
Nova Sphere requires an automation engine to handle asynchronous operational processes like Fraud Review, Refund Pipelines, and Vendor Onboarding. 

## Decision
We decided to implement a **State Machine** pattern (with strict versioning and immutable audit trails) instead of an arbitrary Directed Acyclic Graph (DAG) or third-party orchestration tool (like Temporal).

## Rationale
- E-commerce workflows are predominantly state-driven (Pending -> Approved -> Shipping), making state machines the most natural mental model.
- Avoiding third-party orchestration reduces infrastructure complexity and vendor lock-in.
- Strict versioning (v1, v2) ensures existing orders don't break when operational pipelines are updated.
- Immutable audit trails satisfy compliance requirements out-of-the-box.

## Consequences
- Complex parallel branching (which DAGs excel at) will require custom handling within the state machine.
- We must maintain the workflow engine code natively.
